QGIS is a free, open source, geographic information system software.

Today we will walk through the QGIS interface touching on some of the recent upgrades.

You will want to have QGIS version 3.2 to follow along: QGIS Download

Task 1: Add data

County data was downloaded from the LA County open data site: County Boundary Download

New Feature

Instead of multiple add data buttons, now there is a consolidated Data Source Manager.

Let's add our County Boundaries first. They are a vector data file, so select Vector, find the CountyBoundaries shapefile, & click Add.

Next we will add a WMS tile layer of aerial imagery. Go to the WMS/WMTS tab.

Click New and enter a layer Name and the URL: https://map.dfg.ca.gov/arcgis/services/Base_Remote_Sensing/NAIP_2016/ImageServer/WMSServer? Click ok.

Click Connect and then select the layer and click Add. We're done adding layers for now, so click Close.

Your map now has 2 layers. If the NAIP is above Counties, drag it to the bottom of the Layers list.

Let's apply the local coordinate system to the map. Click on EPSG in the lower right of your QGIS window.

EPSG stands for European Petroleum Survey Group. As a global inventory of spatial reference systems, it is widely used.

New Feature

Search for 2229. Notice the new preview window where you can see the usable extent of that projection.

Click Apply/Ok and return to the map. We'll add a few more layers before we explore further.

Open the Data Source Manager again and go to the Raster tab. Find the 2 .hgt files and click Open & then Add

New Feature

You can now easily connect live to services from an API! We're going to add National Park boundaries direct from the park service.

Check your favorite Open Data portals to see if they have an API that serves standard formats. This way you always have the latest data.

New Feature

Go to the Vector tab and toggle to Protocol. Set the type to GeoJSON & the URL to: https://opendata.arcgis.com/datasets/4e9a3e5a969a451d9c41b2a0b5981b20_0.geojson

Click Add and return to the map. It looks like a mess, but we'll clean that up.

Double click the park layer to open Properties. Go to the Source tab & rename the layer.

Click Query Builder and add a query for Channel Islands park: "PARKNAME" = 'Channel Islands'

New Feature

You'll notice there is now a filter symbol next to your park boundary layer. This let's you know you are working with a data subset.

Go to Plugins, Manage and install Plugins & search for "osm". Install Quick Map Services (basemaps) & QuickOSM (OpenStreetMap data download).

Quick Map Services is found under the Web menu. Add OSM Standard.

Turn off all the layers except your basemap and zoom to the extent of Santa Cruz island.

Click the QuickOSM icon (it looks like a green map). Select key=highway and value=path. Go to Advanced and unselect points & polygons. Click Show query.

We want paths & tracks. We could run 2 quick queries, but let's use regex to get all trails on the island. Change "v" to:
regv=".*"or regv="path|track"

After you click Run query you will have a temporary layer. Right click on the layer, Export, Save feature as.

We're going to save as geopackage. A geopackage is an open data format that uses a SQLite database.

It's a bit like an Esri geodatabase (but open) and much more powerful than a shapefile.

You will enter the geopackage (database) name under File Name (type geopackage) and under Layer Name name your new layer.

New Feature

Notice at the top in the title bar there is an asterisk before your map name. This means you have unsaved changes. Save now.

Task 2: Visualize data

Let's label the counties. Double click County Boundaries in the layers list & select the Label tab.

Under Text, set the Size to 1500 Meters at Scale.

Under Formatting, Multiple Lines, set the Wrap on character to be a space and the Alignment to be center.

Under Placement, tick Offset from centroid. Under Centroid, tick Whole polygon.Check Force point inside polygon.

Under Rendering, Feature Options, check Label every part of multi-part features.

Now we have labels! But we want more control over our labels. The islands are sometimes grouped w/ the mainland.

Go to Vector, Geometry Tools, Multipart to Singleparts.

Select the ellipses next to Singlepart and Save to GeoPackage. Save as a new layer in your geopackage.

New Feature

Click Run in Background. This lets you keep working while a process runs.

Copy the style from your County Boundaries shapefile layer by right clicking on the layer name and selecting Copy style...

Then paste the style by right clicking on your geopackage layer name and clicking Paste style.

Start editing by right clicking on the layer you wish to edit & click Toggle editing.

New Feature

Find the Label toolbar and click on Move label (the arrow icon). Click on a label & you will be prompted to set a unique id.

New Feature

Adjust labels to new positions better suited for your map. The label is still tied to the data, but will show up where you place it.

New Feature

Click on Show/Hide labels (the eye icon). Shift+Click on an island you want to hide the label of (click to bring the label back).

New Feature

We can also have different fonts! Let's make the label for San Miguel island smaller so it fits better.

Turn off all layers except the 2 rasters. Go to Raster, Miscellaneous & select Merge. Save the file as .tif. Now we have one terrain layer (you can remove the originals).

Zoom to the extents of Santa Cruz island and go to Raster, Extraction & select Clip raster by extent. Click the ellipses for Clipping extent and set to Use layer/canvas extent

Now we have a terrain for only the island. Douple click on your clipped terrain to open Properties. On the Symbology tab set Render type to singleband pseudocolor.

Double click the color ramp and you can set the colors however you want. For example, I moved the slider for green so only water shows up as blue.

New Feature

Go to ,View, 3D map view. Click on the wrench & set the Elevation layer to your clipped terrain, Vertical scale to 5, & Map tile resolution to 100px.

New Feature

Use the mouse wheel and click & drag to navigate the 3D view. Tilt by dragging w/ the scroll wheel pressed (or Shift + arrow keys).

Task 3: Analyze data

Go to View, Panels and turn on the Processing Toolbox. A lot of improvements were made to the toolbox and tools in recent releases.

New Feature

Search for Import Geotagged Pictures and point it to the images folder. You will save the result to your geopackage as a Photos layer.

To quickly view the images and highlight the associated point, go to Database, eVis, eVis Event Browser.

Let's do some processing! Go to Processing, Graphical Modeler. We're going to create a model that buffers our trails at different widths.

Under Inputs select Vector layer. Create a parameter named Trails of geometry type line.

New Feature

Under Algorithms search for & select Feature Filter. Click the square with green icon to add 2 outputs, Path & Track. "highway"='path'"highway"='track'

Under Algorithms search for & select Buffer. Set the Description & pick your Path filter as your Input layer.

Do the same thing for your Tracks, but change the Distance to 20. Don't forget to set the Description.

Under Algorithms search for & select Load layer into project. Set the Layer to your Buffer output & give it a Loaded layer name.

Repeat the loaded layer for the tracks buffer. Give the model a Name, Group & then save. Then click the run icon.

We now have 2 buffers in the map! If you search in your Processing toolbox, your custom tool will be available for you to run again.

Let's look at the attrbute table. Right click on Trails and select Open attribute table.

Click on the pencil icon to start editing the table. Then click on the Add field icon. Name will be update and Type will be Date & Time.

New Feature

You can have fields auto-update! Open Trails Properties & go to Attributes Form tab. Set Default value to now() & check Apply default value on update.

We're done w/ the tutorial, but now you should experiment! Modify these tasks to use different data. Then try other tools.
